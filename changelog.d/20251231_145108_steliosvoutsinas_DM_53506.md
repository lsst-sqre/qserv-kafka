### New features

- Add support for multiple database backends through a new `DatabaseBackend` abstraction layer. The bridge can now work with Qserv (existing) or Google BigQuery (new) as the query execution backend, selected via the `backend` configuration parameter.
- Implement `BigQueryClient` backend adapter supporting query submission, monitoring, and result retrieval via the BigQuery REST API.
- Add backend-agnostic progress tracking supporting both chunk-based progress (Qserv) and byte-based progress (BigQuery) through a unified `ProgressMetrics` interface.
- Refactor Qserv implementation into `QservClient` implementing the `DatabaseBackend` interface.
